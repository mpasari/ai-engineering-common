## 7. How to update

To update to the latest version of the commons:

```powershell
# Check what version is available
npm outdated @telia-company/ai-engineering-common

# Update to latest
npm update @telia-company/ai-engineering-common

# Or update to a specific version
npm install @telia-company/ai-engineering-common@{version}

# Regenerate all tool configs
npx aec update

# Review changes in CHANGELOG.md of the commons package
# Commit updated package.json and generated configs
git add package.json package-lock.json .github\copilot-instructions.md CLAUDE.md .cursorrules COMMONS_VERSION.md
git commit -m "chore: update ai-engineering-common to v{version}"
```

## 8. Breaking change guidance

When updating across a major version boundary (e.g. 1.x to 2.x):

```
1. Read the MIGRATION_GUIDE.md in coe/MIGRATION_GUIDES/ for that version
2. Review project-layer override files for compatibility
3. Test the generated tool configs before committing
4. Allow extra time -- major versions may require .ai/project/ file updates
```

Major version migration guides are published to:
`node_modules/@telia-company/ai-engineering-common/coe/MIGRATION_GUIDES/`

## 9. Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| This file shows old version after npm update | aec update not run after npm update | Run `npx aec update` |
| Template variables like {{VERSION}} visible | CLI failed to substitute values | Check CLI version, re-run aec update |
| File is missing entirely | aec init not run | Run `npx aec init` |
| Version does not match package.json | Manual edit or partial update | Run `npx aec update` to regenerate |

## 10. For CoE -- template notes

This file in the commons repository is the template used by the CLI.
The `{{PLACEHOLDER}}` variables are substituted by `scripts/cli.js`
at generation time.

Template variable reference:

| Placeholder | Substituted with |
|---|---|
| `{{COMMONS_VERSION}}` | Value from commons package.json `version` field |
| `{{INSTALL_DATE}}` | ISO 8601 date when `aec init` was first run |
| `{{UPDATE_DATE}}` | ISO 8601 date when `aec update` was last run |
| `{{CLI_VERSION}}` | Version of the aec CLI that generated the file |
| `{{PROJECT_NAME}}` | Value from consuming project's package.json `name` field |
| `{{ARCH_MODIFIED}}` | Last modified date of ARCHITECTURE_OVERVIEW.md |
| `{{MODULE_MODIFIED}}` | Last modified date of MODULE_REGISTRY.md |
| `{{INTEGRATION_MODIFIED}}` | Last modified date of INTEGRATION_MAP.md |
| `{{DATA_MODIFIED}}` | Last modified date of DATA_MODEL.md |
| `{{KAFKA_MODIFIED}}` | Last modified date of KAFKA_TOPICS.md |
| `{{DEBT_MODIFIED}}` | Last modified date of TECH_DEBT_REGISTRY.md |
| `{{ENV_MODIFIED}}` | Last modified date of FEATURE_ENV_CONFIG.md |
| `{{SRE_MODIFIED}}` | Last modified date of SRE_SERVICE_CONFIG.md |
| `{{VERSION_HISTORY}}` | Appended log of previous versions and update dates |

When the CLI writes this file to a consumer project, all placeholders
are replaced with real values. If a project-layer file does not exist,
the corresponding `Last modified` field shows `not present`.
