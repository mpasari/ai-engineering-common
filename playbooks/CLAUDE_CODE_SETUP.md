## 1. Installation

```powershell
# Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Verify
claude --version

# Authenticate with your Anthropic account
claude login
# Opens browser -- sign in with your Anthropic account
# Claude Code is included with Claude Pro and Team plans
```

## 2. First session in your project

```powershell
# 1. Navigate to your project
cd "path\to\your\project"

# 2. Load credentials from .env (see SECRETS_GUIDE.md)
Get-Content .env | Where-Object { $_ -notmatch '^#' -and $_ -ne '' } | ForEach-Object {
    $name, $value = $_ -split '=', 2
    [System.Environment]::SetEnvironmentVariable($name.Trim(), $value.Trim(), 'Process')
}

# 3. Start Claude Code
claude

# You should see:
#   Claude Code (version X.X.X)
#   Loaded: CLAUDE.md (N lines)
#   MCP servers: jira, confluence, github (if configured)
#   >
```

## 3. Verify CLAUDE.md is loaded

At the Claude Code prompt:

```
> What project is this? List the modules and their status.
```

Expected response: Claude describes your service accurately, lists
the modules from MODULE_REGISTRY.md with their Active/Legacy status.

If Claude gives a generic response about software engineering:
- CLAUDE.md is empty or missing
- Run `npx aec update` in another terminal
- Restart Claude Code

## 4. Verify MCP connections

## 5. Running your first command

Start with a low-stakes command to get comfortable:

```
> EXPLAIN_MODULE [one of your module names]
```

Claude Code will:
1. Acknowledge the command
2. Read the relevant files (you see "Reading..." status)
3. Apply the Legacy Explainer Agent skill file from CLAUDE.md
4. Return a structured module analysis

This is a read-only operation -- safe to run any time.

## 6. How Claude Code uses CLAUDE.md

CLAUDE.md (generated by `npx aec update`) contains multiple sections
separated by `---` dividers. Claude Code loads the entire file as context.

The sections are:
```
Section 1: foundation/AGENT.md       -- who Claude is as an agent
Section 2: foundation/HITL_PROTOCOL.md -- gate system it must follow
Section 3: foundation/CODING_STANDARDS.md -- code patterns to use
Section 4: .ai/project/ARCHITECTURE_OVERVIEW.md -- your architecture
Section 5: .ai/project/MODULE_REGISTRY.md -- your modules
Section 6: .ai/project/INTEGRATION_MAP.md -- your integrations
Section 7: .ai/project/DATA_MODEL.md -- your data model
Section 8+: Agent skill files (8-14 depending on your project)
```

When you type `WRITE_SPEC PROJ-412`, Claude Code:
1. Recognises the command from the commands/ directory context
2. Applies the SPEC_WRITER_AGENT.md skill file (loaded in section 8+)
3. Uses your project context (sections 4-7) for project-specific output
4. Calls the Jira MCP to read the story
5. Follows the HITL_PROTOCOL.md gate process

## 7. Key Claude Code behaviours to know

### 7.1 File editing

Claude Code can edit files directly. Before it does:
- It shows you the proposed change as a diff
- You type `yes` to approve or `no` to reject
- You can type `edit` to modify the proposed change

You are always in control of what gets written to disk.

### 7.2 Command execution

Claude Code can run commands (npm test, git commit, etc.):
- It shows you the command before running it
- You type `yes` to allow or `no` to refuse
- Risky commands (delete, production access) require explicit approval

### 7.3 MCP tool calls

When Claude Code calls a Jira or Confluence MCP tool:
- It shows you what it is about to do ("Reading Jira story PROJ-412")
- For writes ("Creating Jira comment"), it shows you the content first
- You can review and approve each write

### 7.4 HITL gates

When an agent command reaches a HITL gate:
- Claude Code shows the formatted gate output
- You are prompted to reply with the gate approval code
- For example: `APPROVED C01` or `CHANGES C01 [feedback]`
- Claude Code only proceeds after your approval

## 8. VS Code integration

Run Claude Code in the VS Code integrated terminal:

```
VS Code:  Ctrl+` (backtick) to open terminal
Terminal: claude (after loading .env)
```

With this setup:
- Claude Code reads and writes files in your workspace
- VS Code shows file changes in the source control panel
- You can review Claude Code's commits in GitLens
- Copilot runs in parallel for inline completion

The two AI tools complement each other:
- Copilot: suggests code as you type in the editor
- Claude Code: executes structured agent workflows in the terminal

```
claude --compact

# See what CLAUDE.md contains

# Force a specific agent

```

## 10. Context window management

Claude Code has a large context window but CLAUDE.md can be 50,000+
tokens for a well-populated project. If you hit context limits:

```
# Check current context size
> How many tokens is your current context?

# Reduce CLAUDE.md size by trimming project files
# Edit .ai/project/ files to be more concise
# Then run: npx aec update

# Use compact mode for large analysis tasks
claude --compact
```

The CONTEXT_WINDOW_STRATEGY.md foundation file guides Claude Code on
how to prioritise loading when context is tight.
