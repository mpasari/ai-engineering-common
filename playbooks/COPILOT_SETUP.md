## 1. Understanding the three-tool setup

You use three tools simultaneously. Each has a different strength:

```
GitHub Copilot (VS Code extension):
  STRENGTH: Inline code completion, Copilot Chat for quick questions
  CONTEXT:  Reads copilot-instructions.md automatically
  BEST FOR: Writing code, explaining code, quick refactors, unit tests
  LIMIT:    No file system access, no Jira/Confluence, no MCP support

Cursor (IDE):
  STRENGTH: Composer mode for multi-file edits, MCP server support
  CONTEXT:  Reads .cursorrules automatically
  BEST FOR: Multi-file refactoring, complex code changes with context
  LIMIT:    Smaller context window than Claude Code, MCP still experimental

Claude Code (terminal inside VS Code):
  STRENGTH: Full agent tasks, MCP access to Jira/Confluence/GitHub
  CONTEXT:  Reads CLAUDE.md (the richest context -- includes agent skill files)
  BEST FOR: WRITE_SPEC, TRIAGE_BUG, VALIDATE_STORY, REVIEW_PR
  LIMIT:    Terminal-based, not inline in editor
```

The typical workflow combines all three:
- Copilot generates code inline as you type
- Cursor refactors across multiple files
- Claude Code handles the agent workflows (specs, tickets, reviews)

## 2. VS Code + GitHub Copilot setup

### 2.1 Install the extensions

Install from VS Code Extensions panel (Ctrl+Shift+X):

```
Required:
  GitHub Copilot           -- publisher: GitHub
  GitHub Copilot Chat      -- publisher: GitHub

Recommended:
  GitHub Pull Requests     -- publisher: GitHub
  GitLens                  -- publisher: GitKraken
```

### 2.2 Sign in to GitHub Copilot

1. Click the Copilot icon in the VS Code status bar
2. Sign in with your GitHub account (must have Copilot licence)
3. Verify: a small Copilot icon appears in the bottom status bar

### 2.3 Configure copilot-instructions.md

The `copilot-instructions.md` file in `.github/` is automatically
loaded by GitHub Copilot as context for every session. This file
is generated by `npx aec update`.

Verify it is loaded:

```
1. Open Copilot Chat (Ctrl+Alt+I or click the chat icon)
2. Type: What project is this and what modules does it have?
3. Expected response: Copilot describes your service using content
   from MODULE_REGISTRY.md

If Copilot gives a generic response:
  -- Run: npx aec update
  -- Check: .github/copilot-instructions.md exists and has content
  -- Restart VS Code
```

### 2.4 How to use Copilot for agent tasks (without MCP)

Since Copilot cannot call Jira or Confluence directly, you paste
context manually. This is less automated than Claude Code but works
if you do not have MCP set up yet.

Pattern for any agent command:

```
1. Open the Jira story in your browser
2. Copy the story description and ACs
3. Open Copilot Chat
4. Type the command + paste the context:

   WRITE_SPEC

   Story: PROJ-412
   Summary: Add order cancellation endpoint

   Description:
   As a customer, I need to cancel an order that is in PENDING status.
   A cancellation reason is required.

   Acceptance criteria:
   Given the user is authenticated with role ORDER_MANAGER
   When POST /api/v1/orders/{id}/cancel is called with a valid reason
   Then the order status changes to CANCELLED

5. Copilot responds using the agent skill file context from
   copilot-instructions.md to generate a technical spec
```

### 2.5 Copilot Chat tips

```
Effective prompts:
  "@workspace What is the architecture of this project?"
  "@workspace How does the order cancellation flow work?"
  "Using the coding standards, generate an OrderController method for..."
  "Review this code for security issues per the checklist:"

Less effective:
  "Write code" (too vague)
  "Fix this" (no context)
  "Is this good?" (no criteria)
```

The `@workspace` prefix tells Copilot to index your entire workspace,
not just the current file. This significantly improves context quality.

## 3. Cursor setup

### 3.1 Install Cursor

Download from https://cursor.sh and install.

Import your VS Code settings on first launch (optional but convenient --
imports extensions, keybindings, and themes).

### 3.2 Configure .cursorrules

The `.cursorrules` file is automatically loaded by Cursor as context.
It is generated by `npx aec update` and contains `CODING_STANDARDS.md`.

Verify Cursor is reading it:

```
1. Open Cursor
2. Press Ctrl+I (Cmd+I on Mac) to open Composer
3. Type: What are the coding standards for this project?
4. Expected: Cursor describes the standards from .cursorrules

If Cursor gives generic advice:
  -- Run: npx aec update
  -- Check: .cursorrules exists and has content
  -- Restart Cursor
```

### 3.3 Configure MCP in Cursor

See MCP_SETUP.md section 6 for full Cursor MCP setup.

### 3.4 Cursor Composer for multi-file tasks

Cursor Composer (Ctrl+I) is most powerful for tasks that touch
multiple files simultaneously:

```
Effective Composer prompts:
  "Implement the cancellation endpoint following the spec in [paste spec URL]
   and the patterns in BACKEND_PATTERNS.md"

  "Refactor all RestTemplate usages in the orders module to use RestClient
   following the pattern in BACKEND_PATTERNS.md section 2.4"

  "Add accessibility attributes to all interactive elements in
   OrderList.tsx following the patterns in FRONTEND_PATTERNS.md"
```

Cursor Composer shows you a diff before applying changes -- always
review the diff before accepting.

## 4. Claude Code setup

### 4.1 Install Claude Code

```powershell
npm install -g @anthropic-ai/claude-code

# Authenticate
claude login
# Follow the browser prompt to sign in with your Anthropic account

# Verify
claude --version
```

### 4.2 Open Claude Code in VS Code terminal

```
1. Open VS Code terminal: Ctrl+` (backtick)
2. Load environment variables (see SECRETS_GUIDE.md section 3.4)
3. Start Claude Code: claude
4. Claude Code opens in the terminal with CLAUDE.md loaded automatically
```

You now have:
- VS Code with Copilot in the editor panels
- Claude Code in the integrated terminal
- Both reading project context from the generated files

### 4.3 CLAUDE.md is the richest context

CLAUDE.md (generated by `npx aec update`) contains:
- AGENT.md -- agent identity and constraints
- HITL_PROTOCOL.md -- gate system
- CODING_STANDARDS.md -- language patterns
- ARCHITECTURE_OVERVIEW.md -- your project's architecture
- MODULE_REGISTRY.md -- your modules
- INTEGRATION_MAP.md -- your integrations
- DATA_MODEL.md -- your entities
- 8+ agent skill files relevant to your project

This means Claude Code understands your project deeply without you
having to explain it. It knows which modules are Legacy, which
integrations need DPA, and which HITL gates apply.

### 4.4 Running agent commands in Claude Code

```
# In Claude Code terminal session:

# Triage a bug
TRIAGE_BUG PROJ-500

# Generate a technical spec
WRITE_SPEC PROJ-412

# Review a PR
REVIEW_PR 142

# Validate story ACs
VALIDATE_STORY PROJ-412

# Explain a module (before touching unfamiliar code)
EXPLAIN_MODULE orders-domain
```

Claude Code reads the CLAUDE.md context, applies the relevant agent
skill file, uses MCP to read/write Jira and Confluence, and executes
the full command workflow.

## 5. The complete daily workflow

### 5.1 Start of day setup

```powershell
# Terminal 1: load environment and start Claude Code
cd your-project
# Load .env
Get-Content .env | ForEach-Object {
    if ($_ -match '^([^#][^=]*)=(.*)$') {
        [Environment]::SetEnvironmentVariable($Matches[1].Trim(), $Matches[2].Trim(), 'Process')
    }
}
claude

# VS Code: open the project
code .
# Copilot activates automatically
# Cursor: open separately if using Composer for multi-file tasks
```

### 5.2 Picking up a Jira story

```
Step 1 (Claude Code): TRIAGE_BUG [key] or confirm story is well-formed
Step 2 (Claude Code): EXPLAIN_MODULE [affected module]  -- understand before coding
Step 3 (Claude Code): WRITE_SPEC [story key]           -- generate tech spec
Step 4 (human): Review spec in Confluence, approve gate C01
Step 5 (Claude Code): GENERATE_CODE [story key]        -- generate code from spec
Step 6 (Copilot): Refine generated code inline in VS Code
Step 7 (Claude Code): REVIEW_PR [PR number]            -- automated peer review
Step 8 (human): Review PR, approve gate D01
Step 9 (Claude Code): VALIDATE_STORY [story key]       -- run AC execution
```

### 5.3 Quick code tasks (Copilot or Cursor only)

For small tasks that do not need a spec or Jira interaction:

```
-- Writing a new method: let Copilot suggest as you type
-- Fixing a lint error: ask Copilot Chat "Fix this lint error"
-- Explaining unfamiliar code: "@workspace explain this class"
-- Multi-file refactor: use Cursor Composer
-- Unit tests for a class: Copilot Chat "Generate unit tests for this class"
```

## 6. Context quality checklist

Run this when AI suggestions feel too generic:

```
[ ] npx aec update was run recently (after project files were filled in)
[ ] .ai/project/MODULE_REGISTRY.md has real content (not stubs)
[ ] .ai/project/ARCHITECTURE_OVERVIEW.md describes the actual system
[ ] CLAUDE.md exists in project root and has > 1000 lines
[ ] .github/copilot-instructions.md exists and has > 200 lines
[ ] .cursorrules exists and has content
[ ] Claude Code was started AFTER loading .env
[ ] VS Code was restarted after copilot-instructions.md was updated
```
