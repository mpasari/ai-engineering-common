## Overview

Cursor is an AI-native IDE built on VS Code. Unlike GitHub Copilot which
bolts AI onto VS Code, Cursor is designed around AI from the ground up.
For our commons workflow, Cursor is the primary IDE for all agent tasks
because:

- Composer (Ctrl+K or Ctrl+I) handles multi-file generation in one session
- MCP servers connect Cursor to Jira and Confluence natively (no terminal needed)
- `.cursorrules` gives the model your coding standards as persistent context
- Agent mode runs sequences of steps autonomously with your approval at each diff
- It reads CLAUDE.md alongside .cursorrules for the richest possible context

## 1. Installation

### 1.1 Download and install

Download from https://cursor.sh (Windows, Mac, or Linux).

On Windows:
```powershell
# Or install via winget
winget install Cursor.Cursor
```

### 1.2 Import VS Code settings (recommended)

On first launch, Cursor offers to import your VS Code configuration:
- Click "Import from VS Code"
- This brings in your extensions, keybindings, themes, and settings
- Copilot extension is not needed in Cursor -- Cursor has its own model

### 1.3 Sign in and select model

1. Open Cursor
2. Cursor > Sign In (top right)
3. Create a Cursor account or sign in
4. Settings > Models > Select: claude-3-5-sonnet (recommended for our workflow)
   - Claude 3.5 Sonnet has the largest context window and best instruction-following
   - GPT-4o is a reasonable alternative

## 2. Configure .cursorrules

Cursor reads `.cursorrules` from the project root as permanent context
for every Composer and Chat interaction. This file is generated by
`npx aec update` and contains `CODING_STANDARDS.md`.

### 2.1 Verify .cursorrules is loaded

Open Cursor in your project, then open Chat (Ctrl+L):

```
What are the coding standards for this project?
```

Expected: Cursor describes the Java/TypeScript patterns, naming
conventions, and commit format from CODING_STANDARDS.md.

If it gives generic advice: run `npx aec update` and restart Cursor.

### 2.2 Add CLAUDE.md as additional context (important)

Cursor reads `.cursorrules` by default but does NOT automatically read
`CLAUDE.md`. To make Cursor aware of your full project context including
agent skill files, add CLAUDE.md as a pinned context file:

**Method 1 -- Reference in .cursorrules (recommended):**

Add this line at the TOP of your `.cursorrules` file (after `npx aec update` generates it):

```
# This file is supplemented by CLAUDE.md which contains the full agent context.
# Always read CLAUDE.md before answering questions about this project.
@CLAUDE.md
```

The `@CLAUDE.md` reference tells Cursor to include it in every session.

**Method 2 -- Pin in Composer:**

In every Composer session, type `@CLAUDE.md` to include the file.
Less convenient but gives you control per session.

**Method 3 -- Cursor rules file (`.cursor/rules`):**

Cursor also reads `.cursor/rules` as persistent project instructions.
Create `.cursor/rules/project.md`:

```markdown
# Project context

This project uses the Telia AI Engineering Common framework.
Always read and follow:
- .cursorrules (coding standards)
- CLAUDE.md (full agent context including architecture and agent skill files)
- .ai/project/MODULE_REGISTRY.md (current module list and status)

When generating code, apply:
- The patterns from BACKEND_PATTERNS.md (section in CLAUDE.md)
- The design tokens from DESIGN_SYSTEM.md (section in CLAUDE.md)
- The HITL gate system from HITL_PROTOCOL.md (section in CLAUDE.md)

Never generate code that violates the "What the agent must never do"
sections of the relevant agent skill files loaded in CLAUDE.md.
```

## 3. MCP server configuration for Cursor

Cursor supports MCP servers via its settings. This is what gives Cursor
access to Jira and Confluence without needing a separate terminal.

### 3.1 Set environment variables (permanent)

Set these as permanent Windows environment variables so Cursor can read
them at startup:

```powershell
# Run once -- survives restarts
[System.Environment]::SetEnvironmentVariable(
    "JIRA_BASE_URL", "https://telia-company.atlassian.net", "User")
[System.Environment]::SetEnvironmentVariable(
    "JIRA_EMAIL", "your.email@telia.com", "User")
[System.Environment]::SetEnvironmentVariable(
    "JIRA_API_TOKEN", "your_jira_token_here", "User")
[System.Environment]::SetEnvironmentVariable(
    "CONFLUENCE_BASE_URL", "https://telia-company.atlassian.net/wiki", "User")
[System.Environment]::SetEnvironmentVariable(
    "CONFLUENCE_EMAIL", "your.email@telia.com", "User")
[System.Environment]::SetEnvironmentVariable(
    "CONFLUENCE_API_TOKEN", "your_jira_token_here", "User")  # same token as Jira
[System.Environment]::SetEnvironmentVariable(
    "GITHUB_TOKEN", "ghp_your_github_token", "User")

# RESTART Cursor after setting these -- it reads env vars on launch
```

### 3.2 Install MCP server packages

```powershell
npm install -g @anthropic-ai/mcp-server-jira
npm install -g @anthropic-ai/mcp-server-confluence
npm install -g @anthropic-ai/mcp-server-github
```

### 3.3 Configure MCP in Cursor settings

Open Cursor Settings (Ctrl+Shift+J):

```json
{
  "cursor.mcp.enabled": true,
  "cursor.mcp.servers": {
    "jira": {
      "command": "npx",
      "args": ["@anthropic-ai/mcp-server-jira"],
      "env": {
        "JIRA_BASE_URL": "${env:JIRA_BASE_URL}",
        "JIRA_EMAIL": "${env:JIRA_EMAIL}",
        "JIRA_API_TOKEN": "${env:JIRA_API_TOKEN}"
      }
    },
    "confluence": {
      "command": "npx",
      "args": ["@anthropic-ai/mcp-server-confluence"],
      "env": {
        "CONFLUENCE_BASE_URL": "${env:CONFLUENCE_BASE_URL}",
        "CONFLUENCE_EMAIL": "${env:CONFLUENCE_EMAIL}",
        "CONFLUENCE_API_TOKEN": "${env:CONFLUENCE_API_TOKEN}"
      }
    },
    "github": {
      "command": "npx",
      "args": ["@anthropic-ai/mcp-server-github"],
      "env": {
        "GITHUB_TOKEN": "${env:GITHUB_TOKEN}",
        "GITHUB_ORG": "telia-company"
      }
    }
  }
}
```

### 3.4 Verify MCP connections

Open Cursor Chat (Ctrl+L) and type:

```
List my open Jira issues in project PARTY
```

Expected: Cursor returns a list of issues from your Jira project.

```
Find the technical spec pages in Confluence for the party-management-service
```

Expected: Cursor lists relevant Confluence pages.

If either fails:
1. Check Cursor > Output > MCP for error messages
2. Verify env vars are set: `$env:JIRA_API_TOKEN` in a new PowerShell
3. Restart Cursor after setting env vars

## 4. Understanding Cursor modes for our workflow

Cursor has three interaction modes. Know which to use when:

### 4.1 Chat (Ctrl+L) -- Questions and analysis

Use Chat for:
- Reading and understanding Jira stories
- Asking about the codebase
- Checking architecture decisions
- Running analysis commands: EXPLAIN_MODULE, TRIAGE_BUG

Chat does not write files. It gives you information and analysis.

```
# Example Chat usage:
EXPLAIN_MODULE party-domain

TRIAGE_BUG PARTY-500

What does the TMF632 Party schema require for Individual party type?

Read Jira story PARTY-42 and summarise the acceptance criteria.
```

### 4.2 Composer (Ctrl+I or Ctrl+K) -- Code generation and editing

Use Composer for:
- Generating new files from a spec
- Making changes across multiple files simultaneously
- Implementing a full story

Composer shows you every proposed file change as a diff.
You approve each change before it is applied.

```
# Example Composer usage:
GENERATE_CODE PARTY-42

Implement the PartyController following the approved spec at [Confluence URL].
Use the patterns in BACKEND_PATTERNS.md for the controller, application,
and domain layers.
```

### 4.3 Agent mode (Cursor 0.43+) -- Autonomous multi-step tasks

Use Agent mode for:
- Long running workflows that span many files and steps
- Commands that need to read Jira, write code, run tests, and iterate

Agent mode runs autonomously and pauses for your approval at key decisions.
It is the closest Cursor gets to the full agent orchestration described
in MULTI_AGENT_SETUP.md.

To enable Agent mode: Composer > toggle "Agent" in the top bar.

```
# Example Agent mode usage:
Acting as the Spec Writer Agent (described in CLAUDE.md),
read Jira story PARTY-42, check CLAUDE.md for the project architecture,
and generate a technical specification following TECHNICAL_SPEC_TEMPLATE.md.
Create the spec as a Confluence page and update the Jira story with the link.
```

## 5. Running agent commands in Cursor

Our 28 command files are prompts designed to be used in any AI tool.
In Cursor they work as Composer or Chat inputs.

### 5.1 How to run a command

**Pattern for commands that write to Jira/Confluence (use Chat with MCP):**

```
# In Cursor Chat (Ctrl+L):
TRIAGE_BUG PARTY-500
```

Cursor reads the bug from Jira (via MCP), checks the KEDB (Confluence via MCP),
enriches the ticket, and updates Jira -- all from the Chat panel.

**Pattern for commands that generate code (use Composer):**

```
# In Cursor Composer (Ctrl+I):
GENERATE_CODE PARTY-42
```

Cursor reads the approved spec from Confluence (via MCP), reads your project
files for context, and generates code across multiple files showing you diffs.

**Pattern for spec generation (use Chat then Composer):**

```
# Step 1: In Chat -- generate the spec content
WRITE_SPEC PARTY-42

# Cursor generates the spec and creates the Confluence page via MCP
# You review and approve gate C01 in Jira

# Step 2: In Composer -- generate the code from the spec
GENERATE_CODE PARTY-42
```

### 5.2 Providing additional context in Composer

When running Composer commands, include file references for the richest output:

```
# Reference specific files Cursor should read:
@CLAUDE.md
@.ai/project/MODULE_REGISTRY.md
@.ai/project/DATA_MODEL.md

GENERATE_CODE PARTY-42

Follow the layer structure in MODULE_REGISTRY.md.
Use the entity definitions in DATA_MODEL.md.
Apply the patterns from BACKEND_PATTERNS.md in CLAUDE.md.
```

The `@filename` syntax tells Cursor to read and include that file in context.

## 6. Cursor settings for our workflow

Add these to Cursor Settings (Ctrl+Shift+J) for the best experience:

```json
{
  "cursor.chat.defaultModel": "claude-3-5-sonnet-20241022",
  "cursor.composer.defaultModel": "claude-3-5-sonnet-20241022",
  "cursor.general.enableShadowWorkspace": true,
  "cursor.terminal.usePreviewBox": true,
  "cursor.chat.showSuggestedFiles": true,
  "cursor.composer.iterationLimit": 20,
  "files.exclude": {
    "node_modules": true,
    ".git": true,
    "target": true,
    "build": true,
    "dist": true
  },
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.organizeImports": "explicit"
  }
}
```

The `enableShadowWorkspace` setting lets Cursor run code and tests in
a sandboxed environment before showing you the results.

## 7. Running the Party Management use case in Cursor

This section walks through the first concrete steps for the Party
Information Management system using only Cursor.

### 7.1 Initial setup (do once per project)

```powershell
# 1. Create the project folder
mkdir party-management-service
cd party-management-service

# 2. Install commons (from GitHub Packages)
npm install @telia-company/ai-engineering-common

# 3. Initialise the project
npx aec init

# 4. Open in Cursor
cursor .
```

### 7.2 Fill in project context (do before sprint 1)

Open each file in `.ai/project/` in Cursor and fill in the content.
Use Cursor Chat to help:

```
# In Cursor Chat:
Help me fill in .ai/project/ARCHITECTURE_OVERVIEW.md for a
TMForum-based Party Management service with Java backend and
React frontend. The service manages B2C, B2B, and B2O party
records following TMF632, TMF669, and TMF644 standards.
Include key constraints around GDPR, soft delete, and
multi-market national ID formats.
```

Review Cursor's suggestion and edit it to match your specifics.

### 7.3 Generate tool configs after filling in project files

```powershell
# In VS Code terminal inside Cursor:
npx aec update

# Verify context quality in Cursor Chat:
# "What project is this and what are its key architectural constraints?"
```

## 9. Troubleshooting

| Problem | Cause | Fix |
|---|---|---|
| Cursor gives generic advice | .cursorrules not loaded | Run npx aec update, restart Cursor |
| MCP not connecting | Env vars not set permanently | Use [System.Environment]::SetEnvironmentVariable with "User" scope, restart Cursor |
| Jira read fails | Token doesn't have project access | Verify token at id.atlassian.com has write access to the project |
| Composer generates wrong patterns | CLAUDE.md not referenced | Add @CLAUDE.md to .cursor/rules/project.md |
| Context limit hit | CLAUDE.md too large | Use @specific-section-file instead of @CLAUDE.md |
| Agent mode not available | Cursor version < 0.43 | Update Cursor: Help > Check for Updates |
| Diffs look wrong | Model misunderstood context | Add more specific @file references and re-run |
