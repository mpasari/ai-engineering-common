## Generate ISO27001_ISMS.md

Save to docs/governance/ISO27001_ISMS.md.

The document uses these status markers throughout:
- COMPLIANT -- evidenced in codebase or PRODUCT_GOVERNANCE.md
- [VERIFY: what to check] -- evidence found, human must confirm
- [GAP-Critical: action] -- missing, blocks certification
- [GAP-High: action] -- must resolve before Stage 2 audit
- [GAP-Medium: action] -- should resolve before certification
- [GAP-Low: action] -- best practice improvement
- [CONFIRMED by Name Date] -- human verified
- [RESOLVED: evidence ref] -- gap fixed
- [N/A: reason] -- not applicable

For each of the 93 controls in Annex A (A.5.1 through A.8.34):
- Assess the control against codebase + PRODUCT_GOVERNANCE.md
- Use industry best practice for controls that cannot be evidenced from code
- Mark non-code controls (HR, physical, governance policies) as [GAP] or [DRAFT] with specific action
- Cloud-hosted physical controls (A.7.1-A.7.14) mark as [N/A: Cloud-hosted -- Azure/provider responsibility]
