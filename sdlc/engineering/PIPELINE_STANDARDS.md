## 1. Required pipeline stages

Every service CI pipeline must include all of these stages in order:

```
1. build          -- Compile and package
2. unit-test      -- Fast tests, no external dependencies
3. integration-test -- Tests with Testcontainers (database, Kafka)
4. security-scan  -- SAST + dependency CVE check + secrets scan
5. lint           -- Code style enforcement
6. sbom           -- Software Bill of Materials generation
7. publish        -- Container image build and push (main branch only)
8. deploy-dev     -- Automatic deployment to dev (main branch only)
```

Stages 1-6 run on every push to every branch.
Stages 7-8 run on push to main or release tag only.

## 2. Standard GitHub Actions workflow

Generated by Pipeline Agent (A29). Reference:

```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: ['**']
  pull_request:
    branches: [main]

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      security-events: write  # for SAST upload to GitHub Security tab
      packages: write         # for image publish

    steps:
      - uses: actions/checkout@v4

      - name: Setup Java
        uses: actions/setup-java@v4
        with:
          java-version: '21'
          distribution: 'temurin'
          cache: maven

      - name: Build
        run: ./mvnw clean package -DskipTests

      - name: Unit tests
        run: ./mvnw test -Dgroups=unit

      - name: Integration tests
        run: ./mvnw test -Dgroups=integration

      - name: Security scan (SAST)
        run: ./mvnw spotbugs:check

      - name: Dependency check
        run: ./mvnw dependency-check:check
        # Fails on CVSS >= 7.0 (High or Critical CVEs)

      - name: Secrets scan
        uses: trufflesecurity/trufflehog@main
        with:
          path: ./
          base: ${{ github.event.repository.default_branch }}
          head: HEAD

      - name: Lint
        run: ./mvnw checkstyle:check

      - name: Generate SBOM
        uses: anchore/sbom-action@v0
        with:
          image: ./target/*.jar
          artifact-name: sbom.spdx
```

## 3. Quality gates

Pipelines fail if any of these thresholds are breached:

| Gate | Threshold | Action on breach |
|---|---|---|
| Unit test pass rate | 100% | Fix failing tests before merge |
| Integration test pass rate | 100% | Fix failing tests before merge |
| SAST critical findings | 0 | Fix or document exception |
| CVE Critical/High | 0 unfixed | Remediate or accept with Security Lead |
| Secrets found | 0 | Rotate credential and fix code |
| Code coverage | >= 60% new code | Add tests or document exception |

## 4. Deployment pipeline

```
dev:     Auto-deploy on merge to main
staging: Auto-deploy on release candidate tag (vX.Y.Z-rc.N)
prod:    Manual trigger after gate A02 approval
```

Production deployment requires:
1. Staging deployment verified (smoke tests pass)
2. Gate A02 approved (Tech Lead + DevOps)
3. Release Agent has generated release notes
4. Deployment runbook is current

## 5. Container image standards

```
Base image: eclipse-temurin:21-jre-alpine (Java)
            node:20-alpine (Node.js)
            mcr.microsoft.com/dotnet/aspnet:8.0-alpine (C#)

Image tagging:
  {registry}/{org}/{service}:{git-sha}   -- immutable, per commit
  {registry}/{org}/{service}:latest      -- mutable, latest main build

Image signing: required for production (cosign)
Image scanning: run Trivy on every built image
```

## 6. Dependency update automation

Dependabot configuration (generated by Pipeline Agent):

```yaml
# .github/dependabot.yml
version: 2
updates:
  - package-ecosystem: maven
    directory: /
    schedule:
      interval: weekly
      day: monday
    open-pull-requests-limit: 10
    ignore:
      - dependency-name: "*"
        update-types: ["version-update:semver-major"]
        # Major version bumps require manual review
    labels:
      - dependencies
      - automated
```
