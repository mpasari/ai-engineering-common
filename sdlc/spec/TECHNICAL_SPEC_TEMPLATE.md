## Spec header (required)

```
Title:          [Feature name] -- Technical Specification
Story:          [PROJ-NNN]
Author:         [Name] / AI Spec Writer Agent (commons v1.0.0)
Status:         Draft -> In Review -> Approved -> Implemented
Created:        [ISO 8601 date]
Last updated:   [ISO 8601 date]
Reviewed by:    [Tech Lead name -- gate C01]
```

---

## 1. Summary

One paragraph. What is being built, why it is being built, and what
problem it solves. Non-technical enough that a product manager can
read it and confirm it matches their intent.

---

## 2. Background and context

Why is this needed now? Link to the product brief, CR ticket, or
customer feedback that motivated the work. What is the current state
and what is the desired future state?

---

## 3. Scope

### 3.1 In scope

Bullet list of what this spec covers and what will be implemented.

### 3.2 Out of scope

Explicit list of what is NOT in scope. This prevents scope creep and
misaligned expectations. Every spec must have this section.

---

## 4. Acceptance criteria

Copy the ACs from the Jira story verbatim. Do not paraphrase.
The spec's ACs are the contract between product and engineering.

```
Given [precondition]
When  [action]
Then  [expected outcome]

[Repeat for each AC]
```

---

## 5. API changes

For each new or modified API endpoint:

```
Method:   POST / GET / PUT / PATCH / DELETE
Path:     /api/v[N]/[resource]
Auth:     Bearer token required / None
Summary:  [One sentence description]

Request body:
  {
    "field": "type -- description -- required/optional"
  }

Response (success):
  HTTP [N]
  {
    "data": {
      "field": "type -- description"
    }
  }

Error responses:
  400 VALIDATION_FAILED -- [When this occurs]
  401 UNAUTHENTICATED  -- [When this occurs]
  403 INSUFFICIENT_PERMISSIONS -- [When this occurs]
  404 NOT_FOUND -- [When this occurs]
  422 BUSINESS_RULE_VIOLATION -- [When this occurs]
```

For no API changes: write "No API changes in this spec."

## 6. Data model changes

For each database change:

```
Table: [table_name]
Change type: ADD COLUMN / CREATE TABLE / ADD INDEX / other

New column:
  Name:     [column_name]
  Type:     [VARCHAR(N) / UUID / NUMERIC(12,2) / TIMESTAMPTZ]
  Nullable: Yes / No
  Default:  [value or none]
  PII:      Yes -- [retention policy] / No

Migration strategy: Direct / Expand-contract
Migration script: V[N]__[description].sql (generated by Data Migration Agent)
```

For no data model changes: write "No data model changes in this spec."

## 8. Security considerations
